Ollama with OpenWebUI on Ubuntu 24.04 LTS

Ollama with OpenWebUI is a self-hosted AI chat platform for Azure, designed to help developers, IT administrators, consultants, and teams deploy a private AI chat environment using Ubuntu 24.04 LTS, Docker, Open WebUI, and Ollama.

What is this?

This offer provides a clean Ubuntu 24.04 LTS virtual machine image with Docker, Docker Compose, Open WebUI, and Ollama configuration prepared for first-run deployment. Open WebUI provides the browser-based interface, while Ollama provides the local model runtime.

No large language model is preloaded in this image. After deployment, customers can choose and pull the Ollama-compatible model that best fits their VM size, licensing requirements, and workload.

First-run instructions

For Marketplace image cleanliness, the application containers are not pre-pulled or started in the captured image. After deploying the VM, complete these steps:

  1. Allow inbound TCP port 3000 in the Azure Network Security Group.
  2. Connect to the VM using SSH.
  3. Start the application service:
sudo systemctl enable --now hoiltd-ollama-openwebui.service

The first startup may take a few minutes while Docker pulls the required Open WebUI and Ollama containers.

After startup, open the web interface:

http://<your-vm-public-ip>:3000

To pull a model, run:

docker exec -it hoiltd-ollama ollama pull llama3.2:3b

Architecture

  • Open WebUI: browser-based chat interface on port 3000
  • Ollama: local model runtime bound to localhost on port 11434
  • Docker Compose: repeatable service management
  • Systemd: controlled startup after deployment

Recommended VM sizing

  • Basic testing: 2 vCPU and 8 GB RAM
  • Recommended start: 4 vCPU and 16 GB RAM
  • Larger models: 8+ vCPU and 32 GB+ RAM, or a suitable GPU-enabled VM
