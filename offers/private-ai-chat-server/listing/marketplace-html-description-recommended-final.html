Private AI Chat Server on Ubuntu 24.04 LTS

Private AI Chat Server is a ready-to-use Azure virtual machine image designed to help teams deploy a self-hosted AI chat environment quickly and consistently. It combines Ubuntu 24.04 LTS, Docker, Open WebUI, and Ollama into a clean server-based deployment that can be launched on Azure without manually assembling the stack.

What is this?

This offer provides a preconfigured private AI chat server. Open WebUI delivers a browser-based chat interface, while Ollama provides the local model runtime. The VM is configured so that the chat interface is available through the web UI, while the Ollama backend is bound locally for improved security.

No large language model is preloaded in this image. After deployment, customers can choose and pull the Ollama-compatible model that best fits their needs, VM size, licensing requirements, and performance expectations.

First-run instructions

For security and Marketplace compliance, the application is not started automatically in the captured image. After deploying the VM, complete the following steps:

  1. Allow inbound TCP port 3000 in your Azure Network Security Group.
  2. Connect to the virtual machine using SSH.
  3. Start the application service:
sudo systemctl enable --now alsourillc-private-ai-chat.service

The first startup may take a few minutes while Docker pulls the required containers.

Once started, access the web interface:

http://<your-vm-public-ip>:3000

Check service status if needed:

sudo systemctl status alsourillc-private-ai-chat.service

To install a model:

docker exec -it alsourillc-ollama ollama pull llama3.2:3b

Recommended VM sizing

VM size depends on the model you choose and the number of users. For a first test deployment, start with a modest general-purpose VM and scale up if needed.

  • Basic testing: 2 vCPU and 8 GB RAM for small models and light usage.
  • Recommended starting point: 4 vCPU and 16 GB RAM for a smoother Open WebUI and Ollama experience.
  • Larger models or heavier use: 8+ vCPU and 32 GB+ RAM, or a suitable GPU-enabled VM where model performance requires it.

Customers should select a VM size appropriate for their chosen Ollama model, expected concurrency, and performance requirements.

What problem does it solve?

Building a private AI chat environment manually can involve installing Docker, configuring containers, exposing ports, securing the backend, and ensuring services persist after reboot. This image removes that complexity by providing a tested, repeatable baseline.

It is especially useful for teams that want to experiment with self-hosted AI chat without committing to complex architectures such as Kubernetes.
